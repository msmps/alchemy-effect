This tutorial provisions a **PlanetScale MySQL** cluster —
Vitess-backed MySQL with the same branching model as PlanetScale
Postgres — and fronts it with **Cloudflare Hyperdrive**, then binds
the connection into your Worker.

Hyperdrive is a Cloudflare-managed pooler that sits between
Workers and your origin database. The Worker sees a familiar
MySQL connection string, but the connection itself is already
pooled at the edge — no per-request TCP handshakes, no cold-start
connection storms.

> Prefer Postgres? See
> [Connect Neon with Hyperdrive](/tutorial/cloudflare/neon-hyperdrive)
> or
> [Connect PlanetScale Postgres with Hyperdrive](/tutorial/cloudflare/planetscale-postgres-hyperdrive).

## Add the PlanetScale provider

`Planetscale.providers()` is its own layer. Register it alongside
`Cloudflare.providers()` in your stack:

```diff lang="typescript"
// alchemy.run.ts
import * as Alchemy from "alchemy";
import * as Cloudflare from "alchemy/Cloudflare";
+import * as Planetscale from "alchemy/Planetscale";
+import * as Layer from "effect/Layer";

export default Alchemy.Stack(
  "MyStack",
  {
-    providers: Cloudflare.providers(),
+    providers: Layer.mergeAll(Cloudflare.providers(), Planetscale.providers()),
    state: Alchemy.localState(),
  },
  // ...
);
```

PlanetScale's auth uses an API token id + secret. `bun alchemy login`
adds a `Planetscale` step that either reads `PLANETSCALE_API_TOKEN_ID`,
`PLANETSCALE_API_TOKEN`, and `PLANETSCALE_ORGANIZATION` from the
environment (good for CI) or stores the credentials under
`~/.alchemy/credentials/<profile>/planetscale-stored.json`.

## Create the cluster

Create `src/Db.ts` with a `Planetscale.MySQLDatabase` — the
long-lived cluster:

```typescript
// src/Db.ts
import * as Planetscale from "alchemy/Planetscale";
import * as Effect from "effect/Effect";

export const Db = Effect.gen(function* () {
  const database = yield* Planetscale.MySQLDatabase("app-db", {
    region: { slug: "us-east" },
    clusterSize: "PS_10",
    allowForeignKeyConstraints: true,
  });

  return { database };
});
```

`clusterSize` picks the underlying hardware tier; `region.slug`
pins where the data sits. `allowForeignKeyConstraints: true`
enables FK enforcement, which Vitess gates behind an explicit
opt-in.

## Fork a branch

```diff lang="typescript"
  export const Db = Effect.gen(function* () {
    const database = yield* Planetscale.MySQLDatabase("app-db", {
      region: { slug: "us-east" },
      clusterSize: "PS_10",
      allowForeignKeyConstraints: true,
    });

+   const branch = yield* Planetscale.MySQLBranch("app-branch", {
+     database,
+     isProduction: false,
+   });

-   return { database };
+   return { database, branch };
  });
```

`MySQLBranch` forks the cluster — cheap to spin up per preview
environment. `isProduction: false` flags this one as
non-production so it skips production safeguards (single-writer
locks, schema-change gates).

## Materialize a password for Hyperdrive

```diff lang="typescript"
    const branch = yield* Planetscale.MySQLBranch("app-branch", {
      database,
      isProduction: false,
    });

+   const password = yield* Planetscale.MySQLPassword("app-password", {
+     database,
+     branch,
+     role: "readwriter",
+   });

-   return { database, branch };
+   return { database, branch, password };
  });
```

`MySQLPassword` materializes a user + password we can feed to
Hyperdrive. The password's `origin` (host / port / database / user
/ password) is materialized lazily so the actual secret never
lands in plan output.

## Put Hyperdrive in front

Hyperdrive needs an `origin` describing where your database lives.
`password.origin` is a pre-parsed `{ scheme, host, port, database,
user, password }` output ready to feed straight in:

```diff lang="typescript"
// src/Db.ts
+import * as Cloudflare from "alchemy/Cloudflare";
import * as Planetscale from "alchemy/Planetscale";
import * as Effect from "effect/Effect";

export const Db = Effect.gen(function* () {
  // ...
  return { database, branch, password };
});

+export const Hyperdrive = Effect.gen(function* () {
+  const { password } = yield* Db;
+  return yield* Cloudflare.Hyperdrive("app-hyperdrive", {
+    origin: password.origin,
+  });
+});
```

Its `scheme` is `"mysql"`, so Hyperdrive provisions a MySQL pooler.
The `origin` is an `Output`, so alchemy orders `MySQLDatabase` →
`MySQLBranch` → `MySQLPassword` → `Hyperdrive` on the deploy graph.

## Bind Hyperdrive at init

`Cloudflare.Hyperdrive.bind(...)` resolves the runtime accessor for
the binding — `connectionString`, `host`, `port`, `user`,
`password`, `database`, plus a `raw` escape hatch. Yield it once at
init:

```diff lang="typescript"
// src/Api.ts
import * as Cloudflare from "alchemy/Cloudflare";
import * as Effect from "effect/Effect";
import * as HttpServerResponse from "effect/unstable/http/HttpServerResponse";
+import { Hyperdrive } from "./Db.ts";

export default class Api extends Cloudflare.Worker<Api>()(
  "Api",
  { main: import.meta.path },
  Effect.gen(function* () {
+    const hd = yield* Cloudflare.Hyperdrive.bind(Hyperdrive);

    return {
      fetch: Effect.gen(function* () {
        return HttpServerResponse.text("ok");
      }),
    };
  }).pipe(Effect.provide(Cloudflare.HyperdriveBindingLive)),
) {}
```

Each accessor field (e.g. `hd.connectionString`) is itself an
`Effect` that requires `RuntimeContext` — it only resolves when a
real request is in flight, never at init.

## Derive a SQL client layer from the binding

Wrap `MysqlClient.layer({ url })` in `Layer.unwrapEffect` so layer
construction is deferred until the Effect it's provided to actually
runs. That's what reads `hd.connectionString` lazily — under the
request's runtime context, not at init.

```diff lang="typescript"
+import * as MysqlClient from "@effect/sql-mysql2/MysqlClient";
import * as Cloudflare from "alchemy/Cloudflare";
import * as Effect from "effect/Effect";
+import * as Layer from "effect/Layer";

  Effect.gen(function* () {
    const hd = yield* Cloudflare.Hyperdrive.bind(Hyperdrive);
+   const SqlLive = Layer.unwrapEffect(
+     hd.connectionString.pipe(
+       Effect.map((url) => MysqlClient.layer({ url })),
+     ),
+   );
```

`MysqlClient.layer` provides both the engine client (`MysqlClient`)
and the generic `SqlClient` tag, so downstream code can stay
engine-agnostic.

## Query inside `fetch` and provide the layer

Yield `SqlClient.SqlClient` inside `fetch` and run queries as
tagged templates. Provide `SqlLive` at the `fetch` boundary — that
placement is what makes the construction lazy, since the layer is
built when the request runs (with `RuntimeContext` in scope):

```diff lang="typescript"
+import * as SqlClient from "effect/unstable/sql/SqlClient";

    return {
      fetch: Effect.gen(function* () {
-       return HttpServerResponse.text("ok");
+       const sql = yield* SqlClient.SqlClient;
+       const rows = yield* sql`SELECT NOW() as now`;
+       return yield* HttpServerResponse.json({ ok: true, rows });
-     }),
+     }).pipe(Effect.provide(SqlLive)),
    };
```

Queries are themselves `Effect`s — you `yield*` them directly, no
`Effect.promise(async () => ...)` wrapper, and errors land in the
typed channel as `SqlError`.

## Wire the resources into the stack

Update `alchemy.run.ts` to yield `Db` and `Hyperdrive` so they
become part of the deploy graph:

```diff lang="typescript"
// alchemy.run.ts
import Api from "./src/Api.ts";
+import { Db, Hyperdrive } from "./src/Db.ts";

export default Alchemy.Stack(
  "MyStack",
  { /* ... */ },
  Effect.gen(function* () {
+    yield* Db;
+    yield* Hyperdrive;
    const api = yield* Api;
    return { url: api.url.as<string>() };
  }),
);
```

## Deploy

```sh
bun alchemy deploy
```

Watch the deploy plan: alchemy creates the MySQLDatabase, then the
branch, then the password, then the Hyperdrive that points at the
password's origin, then the Worker with the Hyperdrive binding
attached. After it completes, hit your Worker URL and you should
see something like:

```json
{ "ok": true, "rows": [{ "now": "2026-05-04T07:53:48.123Z" }] }
```

You're now talking to PlanetScale MySQL from the edge through
Hyperdrive.
