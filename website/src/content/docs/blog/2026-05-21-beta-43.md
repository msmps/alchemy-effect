Two things ship in `v2.0.0-beta.43`. Each gets its own
deep-dive post; this is the short version.

## PlanetScale, branched per PR

`alchemy/Planetscale` is the new provider — `MySQLDatabase` and
`PostgresDatabase`, branches, credentials, and a Drizzle
integration that plugs straight into Cloudflare Hyperdrive. The
cluster is owned by one stage; PR previews fork off it with
`Resource.ref(id, { stage })`.

```typescript
const database = stage.startsWith("pr-")
  ? yield* Planetscale.PostgresDatabase.ref("app-db", {
      stage: `staging-${stage}`,
    })
  : yield* Planetscale.PostgresDatabase("app-db", {
      region: { slug: "us-east" },
      clusterSize: "PS_10",
    });
```

→ Full walkthrough: [**PlanetScale, branched per PR**](/blog/2026-05-21-planetscale).

## Layers — refreshed docs + new guide

The [Layers concept](/concepts/layers) now opens with a
Worker-shaped encapsulation example instead of an abstract
`getJob(id)` helper, and there's a new guide for building one
yourself.

→ Full walkthrough: [**Layers, infrastructure behind a typed interface**](/blog/2026-05-21-infrastructure-layers).
